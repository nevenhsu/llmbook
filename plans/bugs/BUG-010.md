**Bug ID**: BUG-010
**嚴重程度**: P0
**標題**: `/profile` runtime error - event handlers cannot be passed to Client Component props
**狀態**: ✅ Fixed (2026-02-09)

**重現步驟**:
1. 訪問 `/profile` 頁面

**期望結果**: 正常顯示用戶資料和帖子列表

**實際結果**: Runtime error:
```
Event handlers cannot be passed to Client Component props.
<... id=... title=... score={0} commentCount=... boardName=... boardSlug=... authorName=... createdAt=... thumbnailUrl=... onVote={function onVote}>
                                                                                                                                        ^^^^^^^^^^^^^^^^^
If you need interactivity, consider converting part of this to a Client Component.
```

**環境**:
- 瀏覽器: Chrome
- 頁面: `/profile`

**截圖**: N/A

**Console 錯誤**:
```
Error: Event handlers cannot be passed to Client Component props.
    at ... (component stack trace)
```

**根本原因**:
Server Component 無法直接傳遽 event handler (onVote) 给 Client Component。

**修復方向**:
1. 將 PostCard 改為 Client Component
2. 或使用 client directive 包裝需要互動的部分
3. 或重構資料獲取和互動邏輯的職責分配

**已採用方案**:
創建了 `ProfilePostList` Client Component 來處理帖子列表和投票邏輯，避免在 Server Component 中傳遞 event handlers。

**修改檔案**:
- 新增: `src/components/profile/ProfilePostList.tsx`
- 修改: `src/app/profile/page.tsx`
